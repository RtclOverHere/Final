Ros master  เป็น Server สำหรับให้ node หลายๆตัวเชื่อมกันเเละส่งข้อความหากันได้
การทำงานของ ROs topic เเละ Ros Service
Ros Topic เปิดใช้งานการเผยแพร่ข้อความไปยังสมาชิกหลายคนแบบอะ
ซิงโครนัส ทำให้เหมาะสำหรับการสตรีมข้อมูล
Ros Service : การใช้รูปแบบ Request / reply กำหนดการส่งข้อความ โดยสร้าง
ข้อความหนึ่งสำหรับการร้องขอและอีกหนึ่งสำหรับการตอบกลับ เพื่อส่งข้อความที่ร้องขอไป
ยังโหนดที่ร้องขอ
ความแตกต่างของ Topic กับ Message 
Topic: publisher node จะ publish message ของตัวเองขึ้นไปที่ 
topic แล้วจะมี subscriber node มารอรับ message ที่ส่งมาทาง topic ที่มีชื่อเดียวกัน
Topic ROS ทำหน้าที่เป็นช่องทางการสื่อสารหรือบัสโหนดสามารถเผยแพร่ข้อความใน
หัวข้อเฉพาะได้ และโหนดอื่น ๆ สามารถสมัครสมาชิกหัวข้อนั้นเพื่อรับข้อความได้
Message :node แต่ละตัวจะคุยกันผ่าน message, โดย message ก็
อาจจะเป็นชนิดของตัวแปร เช่น integer, floating point, boolean หรืออาจจะเอามา
ประกอบกันเป็น structure เช่นประกอบเป็น array หรือประกอบรวม int, float, string 
เข้าด้วยกัน
ความแตกต่างของเชิงใช้งาน ROS Topic และ ROS Service
Topic ใน ROS เป็นกลไกการเผยแพร่และสมัครสมาชิกสำหรับการสื่อสาร
โดยมีรูปแบบอำนวยความสะดวกในการสื่อสารแบบอะซิงโครนัส หมายความว่า
โหนดผู้เผยแพร่ไม่จำเป็นต้องรอให้ผู้สมัครสมาชิกได้รับข้อความ
: Service บริการใน ROS มีกลไกการตอบสนองคำขอสำหรับการสื่อสาร
อำนวยความสะดวกในการสื่อสารแบบซิงโครนัส หมายความว่าโหนดไคลเอ็นต์ส่งคำ
ขอและรอจนกว่าจะได้รับการตอบกลับจากโหนดเซิร์ฟเวอร์
